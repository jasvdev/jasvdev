# Documentation Skill

When asked to document code or a project, follow these guidelines:

1. **JSDoc/TSDoc Comments**:
   - For functions and methods, provide a description, `@param` tags with types and descriptions, and an `@returns` tag.
   - For complex interfaces or classes, provide a high-level overview of their purpose.
   - Explain _why_ a piece of code exists, not just _what_ it does (if it's not immediately obvious).
   - Flag any side effects.

2. **README Generation/Updates**:
   - Ensure the README has a clear title and description.
   - Include sections for: Installation, Usage, Configuration, and Testing.
   - Use clear formatting with headers, code blocks, and lists.
   - If adding a new feature, update the relevant sections of the README to reflect the change.

3. **Clarity and Tone**:
   - Keep sentences concise.
   - Use an informative and professional tone.
